# 开发方案说明

## 1. 业务目标

用户的工作是负荷预测，经常需要在供需预测模板中根据上级给定的容载比、增长率或容量要求，反推未来年份负荷、容量、同时率或区外送受电应如何调整。

本工具目标是替代人工反复试算。

## 2. 核心业务口径

- 年份不写死，支持 2031、2035 等未来新增年份；
- 2025 年及以前为现状数据，默认锁定；
- 最常见反推目标是容载比和增长率；
- 容载比合理区间通常为 1.3–2.5；
- 配变平均负载率应在 0–1，通常关注 0.5 附近；
- 同时率允许小范围调整，范围 0.83–0.99，单次调整不超过 0.1；
- 项目投产年不可修改；
- 区外送（+）受（-）电可作为低优先级兜底方式；
- 导出文件为新副本，原始传入表格不修改。

## 3. 为什么不做完整 Excel 公式引擎

供需预测模板中存在大量 `SUMIFS`、`INDEX`、`MATCH`、`IFERROR` 等复杂公式。完整复刻 Excel 计算引擎成本高、风险大，也无法表达很多没有写在公式里的业务上下级关系。

因此本工具采用：

```text
Excel 作为输入输出载体
业务模型作为计算核心
```

## 4. 导出策略

导出时：

1. 复制原始工作簿到新文件；
2. 将用户确认的年份调整值写入副本中对应 sheet；
3. 新增 `预测结果表`；
4. 在 `预测结果表` 中记录目标达成、调整项、预测明细和写回清单。
